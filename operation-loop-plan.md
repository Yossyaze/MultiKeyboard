# iPad + iPhone 操作ループ計画書

## 1. 目的
Mac からいつでも開始・停止できるループを作る。

1. iPad に制御を移す（Universal Control）
2. iPad 向けのキーを送る（Switch Control の実行用）
3. 待機する
4. iPhone ミラーリングに制御を移す
5. BetterTouchTool のアクションを実行する
6. 待機する
7. 停止されるまで繰り返す

## 2. 各ツールの役割
- Universal Control: Mac から iPad へポインタとキーボード制御を移す。
- iPad の Switch Control: 受け取ったキーを iPad 側のコマンドに変換する。
- iPhone ミラーリング: iPhone の操作画面を Mac 上のウィンドウとして扱う。
- BetterTouchTool（BTT）: iPhone 側の操作をトリガーキーで実行する。
- Hammerspoon: ループ本体の制御を担当する（開始、停止、待機、繰り返し）。

## 3. 事前条件の確認
- Mac、iPad、iPhone に同じ Apple Account でサインインしていること。
- すべての端末で Wi-Fi、Bluetooth、Handoff が有効であること。
- Mac と iPad の間で Universal Control が手動で動作すること。
- iPhone ミラーリングを Mac から開いて操作できること。
- BTT がインストールされ、必要な権限が付与されていること。
- Hammerspoon がインストールされ、アクセシビリティ権限が付与されていること。

## 4. キーとホットキーの設計（衝突回避）
- ループ開始: Ctrl + Option + Command + S
- ループ停止: Ctrl + Option + Command + X
- iPad 用の操作キー（Switch Control 入力）: F14
- iPhone 用の BTT トリガー: Ctrl + Option + Command + 1

注意:
- iPad 用のキーは F14〜F19 のように、BTT のトリガーキーと分ける。
- 緊急停止用のキーはループ停止専用として確保する。

## 5. iPad 側の設定（Switch Control）
1. iPad で Switch Control を有効にする。
2. 外部スイッチとして F14 を追加する。
3. F14 に、iPad の操作フローで必要なコマンドを割り当てる。
4. Mac から F14 を押して、iPad 側で正しく動作するか手動確認する。

## 6. iPhone 側の設定（BTT + iPhone ミラーリング）
1. iPhone ミラーリングを開き、対象アプリ画面が安定していることを確認する。
2. BTT で Ctrl + Option + Command + 1 をトリガーとして作成する。
3. そのトリガーに、iPhone 側で行いたい操作列（クリック、キー入力など）を割り当てる。
4. 可能であれば、トリガーの適用範囲を iPhone ミラーリングのウィンドウに限定する。
5. 手動で 10 回ほど実行し、安定して動くか確認する。

## 7. Hammerspoon によるループ仕様
ループ 1 周の流れ:
1. iPad に制御を移す（画面端へカーソルを押し込む）
2. F14 を送る
3. T1 秒待つ
4. iPhone ミラーリングのウィンドウにフォーカスを移す
5. BTT のトリガーキーを送る
6. T2 秒待つ
7. 実行中フラグが真なら 1 に戻る

初期値の目安:
- T1 = 2.0 秒
- T2 = 1.5 秒
- フォーカス切替後の待機 = 0.25 秒〜0.40 秒

## 8. 開始・停止の動作
- 開始キーで running=true にしてループを開始する。
- 停止キーで running=false にする。
- 待機を挟む各処理では、続行前に running フラグを確認する。
- 停止は、遅くとも現在の待機が終わるタイミングまでに反映されるようにする。

## 9. 安定化のための対策
- フォーカス切替の直後には短い待機を入れる。
- リトライ回数は多くても 1 回にする。
- 開始・停止時には Hammerspoon の通知やログを表示する。
- 長時間の実行時は macOS のスリープを避ける。

## 10. 検証計画
### フェーズ A: 単体確認
1. iPad 用キーだけを 20 回送って、取りこぼしがないか確認する。
2. iPhone 用 BTT トリガーだけを 20 回送って、取りこぼしがないか確認する。

### フェーズ B: ループ統合確認
1. 10 周回して、操作順が正しいか確認する。
2. 30 周回して、失敗回数を測定する。
3. iPad 待機中と iPhone 待機中の両方で停止キーを押してみる。
4. 停止後すぐに再開できるか確認する。

判定基準:
- 停止が毎回確実に効くこと。
- 誤った端末にキーが送られないこと。
- 30 周回での取りこぼし率が 5% 以下であること。

## 11. 運用手順
### 日常運用
1. 必要な iPad と iPhone の対象アプリまたは画面を開く。
2. iPhone ミラーリングが前面に出せる状態か確認する。
3. 開始ホットキーを押す。
4. 最初の 3 周回は動作を確認する。
5. 作業終了時に停止ホットキーを押す。

### 復旧手順
- iPad への切替に失敗した場合: カーソルを端に押し込み直して 1 回だけ再試行する。
- iPhone 側のトリガーに失敗した場合: ミラーリングウィンドウに再フォーカスして継続する。
- 失敗が連続する場合: ループを停止し、アプリ状態をリセットして再開する。

## 12. 今後の拡張案
- キーや待機時間が異なる複数のループ設定を作る。
- 待機時間にランダムな揺らぎを入れて、固定周期を避ける。
- 周回数や失敗回数を記録する簡易ログを追加する。
- 完全停止とは別に、一時停止と再開のキーを追加する。
